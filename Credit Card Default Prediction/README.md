# Credit Card Default Prediction

A beginner-level binary classification project that predicts whether a credit-card client will default on next month's payment, using the **UCI Default of Credit Card Clients** dataset.

## Problem Statement

Given 23 demographic and credit-history features for 30,000 Taiwanese credit-card clients, predict the binary target `default.payment.next.month` (1 = default, 0 = no default).

## Dataset

- **Source**: [Default of Credit Card Clients — UCI / Kaggle](https://www.kaggle.com/datasets/uciml/default-of-credit-card-clients-dataset)
- **Samples**: 30,000 clients
- **Features**: 23 numeric / categorical predictors

| Feature | Description |
|---|---|
| LIMIT_BAL | Credit limit in NT dollars |
| SEX | 1 = male, 2 = female |
| EDUCATION | 1 = grad school, 2 = university, 3 = high school, 4 = others |
| MARRIAGE | 1 = married, 2 = single, 3 = others |
| AGE | Age in years |
| PAY_0..PAY_6 | Repayment status the past 6 months (-2 = no consumption … 8+ = severely delayed) |
| BILL_AMT1..BILL_AMT6 | Bill statement amount over the past 6 months |
| PAY_AMT1..PAY_AMT6 | Amount paid over the past 6 months |
| **default** | **0 = no default, 1 = default** |

## Results

7 classifiers + 1 tuned variant trained and compared. F1 score is the primary metric because the target is moderately imbalanced (~22% default).

| Model | Accuracy | Precision | Recall | F1 Score |
|---|---|---|---|---|
| **Gradient Boosting (Tuned)** | **0.8187** | **0.6648** | **0.3632** | **0.4698** |
| Gradient Boosting | 0.8187 | 0.6648 | 0.3632 | 0.4698 |
| Naive Bayes | 0.6797 | 0.3692 | 0.6330 | 0.4664 |
| Random Forest | 0.8105 | 0.6240 | 0.3602 | 0.4568 |
| Decision Tree | 0.8072 | 0.6098 | 0.3557 | 0.4493 |
| KNN (k=7) | 0.7998 | 0.5829 | 0.3338 | 0.4245 |
| Logistic Regression | 0.8068 | 0.6342 | 0.2992 | 0.4066 |
| SVM (linear) | 0.2385 | 0.1952 | 0.7822 | 0.3124 |

Best GridSearchCV parameters: `{learning_rate: 0.1, max_depth: 3, n_estimators: 100}`

## Key Findings

- **Recent payment status is the dominant signal** — `PAY_0` (most recent month's repayment status) has the strongest correlation with default; clients who delay even one month are far more likely to default.
- **Class imbalance matters**: ~22% of clients default, so accuracy alone is misleading. Naive Bayes had the best recall (0.63) at the cost of precision; Gradient Boosting offers the best balance.
- **Engineered features helped**: aggregate `NUM_DELAYED_MONTHS`, `MAX_PAY_STATUS`, and `BILL_TO_LIMIT_RATIO` ranked among the top predictors in the tree-based models.
- **SVM with a linear kernel did not converge** in 2,000 iterations on this dataset — kernel SVMs are also impractical at 30k rows. Tree-based ensembles dominate.
- **Demographics matter less than behaviour**: SEX, EDUCATION, MARRIAGE add only marginal signal once payment history is included.
